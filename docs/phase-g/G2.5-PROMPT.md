## The architecture, for clarity

The Laravel app at this repo is the **agent system**. It will be deployed to its own custom domain (e.g. `kemeragent.com`). It hosts:

- Admin panel (`/admin`)
- Public agents page (`/`)
- The embed widget file (`/embed/embed.js`)

Operators of the betting site (kemerbet.com or partners) paste this snippet on their site:

```html
<div id="kemerbet-agents"></div>
<script src="https://kemeragent.com/embed/embed.js"></script>
```

The `https://kemeragent.com` part is whatever Laravel's `APP_URL` env var resolves to. Dev shows `http://localhost:8001`, staging shows the staging domain, production shows the real domain.

### Change 2: Backend — expose `APP_URL` via Settings response

The Settings page already calls `GET /api/admin/settings` which returns the 6 user-configurable settings. We need to add the embed base URL to the response so the frontend can build the snippet, but it's NOT a user-editable setting — it's read from `APP_URL` config.

In `app/Http/Controllers/Admin/SettingController.php`, modify `index()` and `update()` to wrap the response shape:

**Before** (current G2 code):
```php
public function index(): JsonResponse
{
    return response()->json(['data' => $this->allSettings()]);
}

public function update(UpdateSettingsRequest $request): JsonResponse
{
    $validated = $request->validated();
    DB::transaction(function () use ($validated) {
        foreach ($validated as $key => $value) {
            Setting::updateOrCreate(
                ['key' => $key],
                ['value' => $value, 'updated_at' => now()],
            );
        }
    });
    Cache::forget('settings.public');
    return response()->json(['data' => $this->allSettings()]);
}

private function allSettings(): array
{
    return Setting::all()->pluck('value', 'key')->all();
}
```

**After:**
```php
public function index(): JsonResponse
{
    return response()->json(['data' => $this->buildResponseData()]);
}

public function update(UpdateSettingsRequest $request): JsonResponse
{
    $validated = $request->validated();
    DB::transaction(function () use ($validated) {
        foreach ($validated as $key => $value) {
            Setting::updateOrCreate(
                ['key' => $key],
                ['value' => $value, 'updated_at' => now()],
            );
        }
    });
    Cache::forget('settings.public');
    return response()->json(['data' => $this->buildResponseData()]);
}

private function buildResponseData(): array
{
    return [
        'settings' => $this->allSettings(),
        'embed_base_url' => rtrim(config('app.url'), '/'),
    ];
}

private function allSettings(): array
{
    return Setting::all()->pluck('value', 'key')->all();
}
```

**This is a breaking change to the API contract.** Response shape changes from:
- Before: `{ data: { prefill_message: "...", agent_hide_after_hours: 12, ... } }`
- After: `{ data: { settings: { prefill_message: "...", agent_hide_after_hours: 12, ... }, embed_base_url: "https://..." } }`

The frontend SettingsPage.tsx must be updated to read `data.settings.X` instead of `data.X`.

### Change 1: Add Install Embed panel to SettingsPage.tsx

Add a new panel positioned **between** the "Telegram Pre-filled Message" panel and the "Agent Behavior" panel. Visible only in the General tab (inside the existing `activeTab === 'general'` branch).

Add new state at top of component:

```ts
const [copyState, setCopyState] = useState<'idle' | 'copied'>('idle');
```

Add the copy handler near the other handlers:

```ts
async function handleCopySnippet() {
    const snippet = `<div id="kemerbet-agents"></div>\n<script src="${embedBaseUrl}/embed/embed.js"></script>`;
    try {
        await navigator.clipboard.writeText(snippet);
        setCopyState('copied');
        setTimeout(() => setCopyState('idle'), 2000);
    } catch {
        // Older browsers / restricted environments — silently fail
    }
}
```
